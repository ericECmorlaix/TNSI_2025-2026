Réseau & protocoles de routage

Infrastructure réseau

Un réseau repose sur deux équipements clés :

  • Switch (commutateur) : relie des machines appartenant au même réseau local ; il travaille avec les adresses MAC.
  • Routeur : relie des réseaux (locaux ou non) d'adresses réseau différentes ; il travaille avec les adresses IP. Il possède au moins deux interfaces, chacune avec une adresse IP propre à son réseau.

L'adresse MAC (Medium Access Control) est l'identifiant physique d'une carte réseau : codé sur 48 bits, notés en hexadécimal (ex. 5E:FF:56:A2:AF:15). Elle est censée être unique et permanente.

À retenir

  • Même réseau → communication directe via le switch.
  • Réseaux différents → passage obligatoire par un routeur.
  • L'adresse IP de destination ne change pas tout au long du trajet ; l'adresse MAC, elle, change à chaque saut entre routeurs.

Adressage IP (IPv4)

Structure d'une adresse

Une adresse IPv4 est codée sur 32 bits (4 octets), en notation décimale pointée : a.b.c.d (chaque octet prend une valeur entre 0 et 255).

Elle se compose de deux parties :

┌─────────────────────┬─────────────────────┐
│    Partie RÉSEAU    │    Partie HÔTE      │
│  (identifie le      │  (identifie la      │
│   réseau/sous-rés.) │   machine)          │
└─────────────────────┴─────────────────────┘

Le masque de réseau indique la frontière : des 1 pour la partie réseau, des 0 pour la partie hôte.
Adresse réseau = IP AND masque (ET logique bit à bit).

IP     : 192.168.181.42  → 11000000.10101000.10110101.00101010
Masque : 255.255.224.0   → 11111111.11111111.11100000.00000000
AND      ─────────────────────────────────────────────────────
Réseau : 192.168.160.0   → 11000000.10101000.10100000.00000000

Notation CIDR

adresse_IP / n où n = nombre de bits à 1 dans le masque. Ainsi, pour l'adresse 192.168.181.42 avec le masque 255.255.224.0, on écrira 192.168.181.42/19.

Les masques /8, /16 et /24 sont les plus simples, car ils « coupent » l'adresse IP pile sur un octet, ce qui rend l'adresse réseau triviale à déterminer.

Notation Masque Machines max dans le réseau
/8 255.0.0.0 2²⁴ − 2 ≈ 16,7 millions
/16 255.255.0.0 2¹⁶ − 2 = 65 534
/24 255.255.255.0 2⁸ − 2 = 254

Adresses réservées

  • Adresse réseau : partie hôte = 0…0 → identifie le réseau, non attribuable.
  • Broadcast : partie hôte = 1…1 → envoi à toutes les machines du réseau, non attribuable.
  • Ces deux adresses ne peuvent pas être assignées à une machine.
IPv6

Les adresses IPv4 (≈ 4,3 milliards) sont épuisées depuis 2012. IPv6 utilise 128 bits, notés en hexadécimal (ex. 2001:db8:0:85a3::ac1f:8001), offrant 2¹²⁸ ≈ 3,4 × 10³⁸ adresses. Ce nombre est colossal : chaque mm² (émergé ou immergé) de la planète Terre peut se voir attribuer environ 667 millions de milliards d'adresses IPv6 ! La transition IPv4 → IPv6 est progressive.

Couche Liaison

C'est typiquement là qu'on trouve les protocoles Ethernet ou Wi-Fi. Une trame Ethernet comporte les adresses MAC destination et source : ces adresses changent à chaque traversée de routeur. Il y a une excellente fiabilité de la communication grâce au FCS (Frame Check Sequence).

Couche Internet

C'est la couche qui assure l'adressage et le routage des paquets d'un réseau à l'autre. Le protocole central est IP (Internet Protocol) : il encapsule les segments TCP/UDP dans des paquets, en leur ajoutant les adresses IP source et destination ainsi qu'un TTL (Time To Live) décrémenté de 1 à chaque routeur traversé — un paquet dont le TTL tombe à 0 est détruit, ce qui évite qu'il circule indéfiniment.

ICMP (Internet Control Message Protocol) sert aux messages de diagnostic réseau (c'est lui qui répond à la commande ping).

ARP (Address Resolution Protocol) permet de retrouver l'adresse MAC correspondant à une adresse IP sur un réseau local.

Couche Transport

TCP garantit la fiabilité : accusés de réception, retransmission en cas de perte, remise en ordre des segments.

UDP vise l'efficacité : les données sont envoyées et… c'est tout ! On emploie UDP lorsque la rapidité prime sur la précision (DNS, flux vidéos streamés, certains jeux…).

Chaque service joignable via TCP ou UDP est identifié par un numéro port standardisé (ex. : SSH ↔ 22, DNS ↔ 53, HTTP ↔ 80, HTTPS ↔ 443). Il y a 65 536 ports différents : de 0 à 1023 on a les ports « bien connus » (utilisés par les principaux services réseaux) ; de 1024 à 49 151 on trouve les ports réservés. Une application client, cherchant à joindre une application serveur via TCP obtient pour cette communication un numéro de port haut, tiré au hasard dans la plage 49 152 – 65 535.

Couche Application

C'est la couche la plus haute : elle regroupe tous les protocoles directement utilisés par les applications. Elle ne s'occupe ni d'adressage ni de routage — elle suppose que la livraison est assurée par les couches inférieures, et se concentre sur le format et la sémantique des échanges.

Parmi les protocoles courants :

  • DNS (Domain Name System) : résolution de noms de domaine en adresses IP) ;
  • FTP (File Transfer Protocol) : transfert de fichiers ;
  • SMTP (Simple Mail Transfer Protocol) : envoi de courrier électronique ;
  • SSH (Secure SHell) : connexion distante sécurisée.
  • HTTP (HyperText Transfer Protocol) et son pendant sécurisé HTTPS : utilisés pour le web (voir la fiche dédiée aux technologies web).

Tables de routage

Rôle et structure

Tout équipement réseau possède une table de routage. Elle liste, pour chaque réseau de destination connu, la façon d'y acheminer un paquet.

Une entrée typique contient :

Destination (réseau) Masque Passerelle (next hop) Interface Métrique
192.168.2.0 /24 10.0.0.1 eth0 1
0.0.0.0 /0 10.0.0.254 eth0 —
  • Destination : toujours un réseau (adresse réseau + masque), jamais un routeur.
  • Passerelle (next hop) : adresse IP du routeur suivant à qui confier le paquet (vide si le réseau est directement connecté).
  • Interface : carte réseau par laquelle le paquet va sortir.
  • Métrique : coût du chemin (plus elle est faible, plus le chemin est préféré).

Destination = réseau, pas routeur

Certains anciens sujets de bac indiquent des routeurs (ex. « R2 ») comme destinations : c'est un abus de langage. La destination correcte est le réseau auquel appartient (au moins) une interface du routeur concerné.

Métrique ?

La métrique sert à mesurer le « coût d'utilisation » d'un chemin vers une destination, en lui attribuant une valeur numérique. Plus la métrique est faible, meilleur est le chemin.

La métrique n'a pas de définition universelle : chaque protocole choisit ce qu'il mesure (voir plus loin). Mais la métrique est toujours une valeur à minimiser : entre deux chemins vers la même destination, le routeur choisit systématiquement celui dont la métrique est la plus faible.

RIP compte le nombre de sauts (liaisons réseau traversées) ; OSPF utilise un coût, inversement proportionnel au débit ; d'autres protocoles peuvent tenir compte de la charge, du délai, de la fiabilité, etc.

Ordinateur vs routeur

Routage direct vs indirect

Avant d'envoyer un paquet, toute machine (ordinateur ou routeur) détermine si la destination est sur le même réseau qu'elle ou non. Elle compare pour cela son adresse réseau avec celle de la destination (en appliquant son masque à l'adresse IP de destination).

  • Les adresses réseau sont identiques → routage direct. Les deux machines sont sur le même réseau local. La communication se fait directement via le switch, sans passer par un routeur.

    Pour la culture : la machine source utilise le protocole ARP pour résoudre l'adresse MAC de la destination. Elle lui envoie ensuite la trame directement.

  • Les adresses réseau sont différentes → routage indirect. La destination est sur un autre réseau, la machine source ne peut pas l'atteindre directement. Elle envoie alors le paquet à sa passerelle par défaut (default gateway) : c'est le routeur connecté au réseau local de la machine source, dont l'adresse IP est indiquée dans la table de routage de la machine. C'est ce routeur qui prend en charge l'acheminement vers la destination finale.

La passerelle par défaut est indispensable !

Sans passerelle par défaut configurée, une machine est incapable de communiquer avec n'importe quel réseau différent du sien. C'est l'entrée 0.0.0.0/0 dans la table de routage : elle correspond à toute destination non couverte par une entrée plus spécifique.

les IP ne changent (presque) jamais !

Les paquets IP forgés par la source pour être transmis à la destination conservent tout le long du chemin les mêmes adresses source et destination. C'est la comparaison entre l'adresse de destination et l'adresse de la machine traitant ces paquets qui permet de savoir comment les orienter.

Pourquoi le « (presque) » alors ? Une technologie nommée NAT (Network Address Translation) permet le passage de communications entre réseaux d'adresses publiques et réseaux privés. Elle implique à cette occasion une réécriture (ou traduction) des adresses IP.

Routage statique vs dynamique

  • Statique : chaque entrée est renseignée manuellement par l'administrateur. Simple, mais incapable de s'adapter aux pannes, et à réserver aux petits réseaux qui n'évoluent pas (ou peu).
  • Dynamique : les routeurs échangent des informations entre eux et mettent à jour leurs tables automatiquement → protocoles RIP et OSPF.

Protocoles de routage dynamique

En bref

Deux protocoles sont à connaître : RIP et OSPF.

  • RIP : chaque routeur demande à son voisin « c'est dans quelle direction et à quelle distance ? » sans chercher à comprendre pourquoi.
  • OSPF : chaque routeur collecte le plan complet du réseau et calcule lui-même le meilleur chemin.

RIP (Routing Information Protocol)

RIP est un protocole à vecteur de distance. Cela signifie que chaque routeur n'a pas la « carte » complète du réseau (on parle de topologie) : il ne dispose que les informations communiquées par ses voisins. Cela lui permet de calculer à quelle distance se trouve chaque destination et par quelle interface réseau l'atteindre au moindre coût.

Un vecteur est un couple (direction, distance) : la direction est l'interface (ou la passerelle) à emprunter, la distance est la métrique (voir plus haut et plus bas).

RIP adopte la même logique que l'automobiliste « à l'ancienne » qui, sur la route, suit les poteaux indicateurs (et pas le GPS) : un routeur aussi « suit la flèche », sans savoir ce qu'il y a au-delà. Lorsque R1 dit à R2 « je peux atteindre le réseau X en 3 sauts », R2 le croit sur parole — il ne sait pas par quels routeurs passe R1 (« routing by rumor »).

Les tables de routage complètes sont échangées périodiquement entre un routeur et ses seuls voisins immédiats (toutes les 30 secondes pour RIP), même si rien n'a changé.

Fonctionnement :

  1. Au démarrage, chaque routeur ne connaît que ses réseaux directement connectés, avec une métrique de 1.

  2. Toutes les 30 secondes, chaque routeur envoie sa table à ses voisins directs.

  3. À réception d'une table voisine, un routeur met à jour la sienne :

    • Nouvelle destination inconnue → ajout d'une entrée dans la table de routage locale, avec pour métrique la valeur reçue augmentée de 1.
    • Destination déjà connue, mais avec une meilleure métrique → mise à jour de la table de routage locale.
    • Destination connue, mais avec un chemin plus long → information ignorée.

  4. Le processus se répète jusqu'à convergence (tables stables).

  5. Si un routeur voisin ne répond plus depuis 3 minutes, il est déclaré en panne : ses routes voient leurs métriques mises à 16 (= infini).

Métrique RIP : nombre de sauts (nombre de liaisons réseau traversées).

Métrique des réseaux directement connectés : RFC vs sujets de bac

  • Selon la RFC 2453 (spécification officielle de RIP) : la métrique d'un réseau directement connecté est 1. Les métriques valides vont de 1 à 15 ; 16 signifie « inaccessible » (infiniment éloigné). Une métrique de 0 est invalide selon le protocole.

  • Dans de nombreux sujets de bac (et dans l'implémentation Cisco) : la métrique des réseaux directement connectés est notée 0, et celle du premier saut à 1. Il s'agit d'un décalage constant qui ne change pas l'ordre des routes.

  • Conduite à tenir à l'examen : suivre ce que dit le sujet. S'il donne 0 pour les réseaux directement connectés, appliquer cette convention. Il est possible (et probablement non-pénalisant) de le signaler sur la copie.

Limites de RIP

  • Maximum 15 sauts : RIP est inutilisable sur les grands réseaux.
  • Ne tient pas compte du débit des liaisons (un lien 1 Gbps et un lien 56 kbps valent tous les deux « 1 saut »).
  • Convergence lente des tables de routage.
Algorithme sous-jacent (hors programme)

RIP s'appuie sur l'algorithme de Bellman-Ford (1956), qui calcule les plus courts chemins dans un graphe pouvant contenir des poids négatifs. Dans le contexte de RIP, le graphe a pour sommets les routeurs et pour arêtes les liaisons réseau (de coût 1).

OSPF (Open Shortest Path First)

OSPF est un protocole à état de lien : contrairement à RIP, chaque routeur acquiert une connaissance complète de la topologie du réseau.

Un lien (link) désigne une interface de routeur et sa connexion à un réseau ou à un voisin. L'état de ce lien, c'est l'ensemble de ses caractéristiques : connecté à qui, avec quel débit, et à quel coût.

Chaque routeur diffuse à tout le réseau une description de ses propres liens (un LSA, Link State Advertisement). Tous les routeurs reçoivent tous les LSA de tous les autres, et chacun reconstruit ainsi une carte complète et identique de la topologie. Les mises à jour ne sont envoyées qu'en cas de changement (et non périodiquement), ce qui explique la convergence rapide.

Fonctionnement pas à pas

  1. Chaque routeur découvre ses voisins et leur envoie des messages LSA décrivant ses liaisons.
  2. Ces LSA se propagent à tous les routeurs du réseau : chaque routeur reconstruit la carte complète.
  3. Chaque routeur calcule individuellement le plus court chemin vers chaque destination.
  4. En cas de panne, la mise à jour est immédiate (pas d'attente de 30 secondes).

La métrique OSPF est un coût, inversement proportionnel au débit de la liaison :

$$\text{coût} = \frac{10^8}{\text{débit } d \text{ (en bits/s)}}$$

Cette formule a toujours (jusqu'en 2026 au moins) été redonnée dans les sujets écrits de bac NSI.

Technologie Débit Coût
Ethernet 10 Mb/s 10
Fast Ethernet 100 Mb/s 1
Gigabit Ethernet 1 Gb/s 0,1
Fibre (10 Gb/s) 10 Gb/s 0,01

L'objectif est de minimiser la somme des coûts sur le chemin.

Coûts décimaux : théorie vs sujets de bac

La formule $\text{coût} = \frac{10^8}{d}$ produit des valeurs décimales pour les débits $d$ supérieurs à 100 Mb/s. En pratique, certaines implémentations concrètes, notamment chez Cisco, utilisent des entiers. Mais la plupart des sujets de bac utilisent des coûts décimaux, conformes à la formule théorique.

TOUJOURS appliquer la convention du sujet !

Avantages sur RIP

  • Pas de limite de taille de réseau.
  • Tient compte du débit réel des liaisons.
  • Convergence rapide.
  • Connaissance globale de la topologie → moins sensible aux boucles de routage.
Algorithme sous-jacent (hors programme)

OSPF s'appuie sur l'algorithme de Dijkstra (1959), qui calcule le plus court chemin depuis un sommet source vers tous les autres sommets d'un graphe à poids positifs.

Lire et remplir une table de routage (méthode)

Dans les exercices de bac, on représente souvent le réseau sous forme de graphe : les sommets sont les routeurs, les arêtes sont les liaisons réseau (avec leur débit ou coût pour OSPF).

Avec RIP : chercher le chemin avec le moins de sauts

  1. Modéliser le réseau en graphe (routeurs = sommets, liaisons = arêtes de poids 1).
  2. Trouver le chemin le plus court en nombre d'arêtes.
  3. La métrique de la table = nombre de sauts (liaisons) depuis ce routeur jusqu'à la destination.

Avec OSPF : chercher le chemin de coût minimal

  1. Calculer le coût de chaque liaison avec $\text{coût} = \frac{10^8}{d}$.
  2. Trouver le chemin minimisant la somme des coûts.
  3. En cas d'égalité, les deux chemins sont équivalents ; le sujet précise généralement lequel retenir.

Que mettre dans la colonne « passerelle » ?

La passerelle est un routeur voisin, le prochain routeur sur le chemin (le next hop), pas la destination finale ni le chemin complet.

Exemple : si R1 doit atteindre le réseau 10.0.3.0/24 en passant par R2 puis R3, la passerelle inscrite dans la table de R1 est R2 (son adresse IP sur leur liaison commune), pas R3.
